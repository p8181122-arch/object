# 五、程式介紹(含 UML 圖、操作說明)：
### 程式模擬撲克牌21點的遊戲規則，在程式碼中包含了洗牌、發牌、計算點數和輸贏判斷機制，我們透過物件導向的架構設計，將現實中的卡牌、牌堆、玩家與遊戲流程抽象化為程式中的「類別 (Class)」。



### 牌局管理功能：自動產生52張撲克牌，並具有隨機洗牌與發牌功能，確保每張牌在同一局中只會出現一次。
### 動態點數計算：能夠精準計算手牌總和。特別針對撲克牌中的 A 實作了動態判定功能，會根據當前總和，自動將 A 視為 1 點或 11 點，以最有利於持牌者的方式去計算。
### 玩家互動介面：透過終端機接收玩家指令，執行「要牌」與「停牌」。
### 莊家決策機制：內建自動化莊家邏輯，依21點規則進行，當莊家點數未滿17點必須要牌。
### 狀態與結果顯示：即時更新並顯示玩家及莊家手牌狀態，並在局末準確判定輸贏。
