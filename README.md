# 五、系統功能說明
程式模擬撲克牌21點的遊戲規則，在程式碼中包含了洗牌、發牌、計算點數和輸贏判斷機制，我們透過物件導向的架構設計，將現實中的卡牌、牌堆、玩家與遊戲流程抽象化為程式中的「類別 (Class)」。


## 功能說明：
### ．牌局管理功能：
自動產生52張撲克牌，並具有隨機洗牌與發牌功能，確保每張牌在同一局中只會出現一次。

### ．動態點數計算：
能夠精準計算手牌總和。特別針對撲克牌中的 A 實作了動態判定功能，會根據當前總和，自動將 A 視為 1 點或 11 點，以最有利於持牌者的方式去計算。

### ．玩家互動介面：
透過終端機接收玩家指令，執行「要牌」與「停牌」。

### ．莊家決策機制：
內建自動化莊家邏輯，依21點規則進行，當莊家點數未滿17點必須要牌。

### ．狀態與結果顯示：
即時更新並顯示玩家及莊家手牌狀態，並在局末準確判定輸贏。

# 七、程式使用方式：
### ．開始遊戲： 
編譯並執行主程式後，系統自動發放初始手牌。 

### ．查看手牌： 
畫面會顯示您的兩張明牌與目前總點數，同時顯示莊家的一張明牌與一張暗牌。 

### ．進行決策： 
系統會提示您輸入選項：請輸入 H 代表要牌 ，或輸入 S 代表停牌 。 若輸入 H，系統會發給您一張新牌並更新畫面。您可以不斷要牌直到您滿意或不小心爆牌為止。 若輸入 S，您的回合結束。 

### ．觀看結果： 
您停牌後，系統會自動翻開莊家暗牌，並快速展現莊家的補牌過程。最後畫面會印出雙方最終點數，以及「You Win!」、「You lose!」或「Tie!」的對戰結果。 

# 八、程式如何安裝執行(圖文講解)：
###  ．依賴環境
* **作業系統：** Windows 10 / 11
* **開發環境：** Visual Studio Code (VS Code)
* **編譯器：** MinGW-w64 (GCC 4.9.2 或以上，可使用 Dev-C++ 內建的 `bin` 檔案路徑)
* **C++ 標準：** C++11 或以上標準 (`-std=c++11`)

###  ．編譯方式
請開啟 VS Code 的內建終端機（Terminal），並確認目前的執行路徑位於專案資料夾內（例如：`PS C:\21點>`）。接著輸入以下指令，將專案中的多個 `.cpp` 原始碼檔案編譯成單一執行檔：

```bash
g++ main.cpp Card.cpp Deck.cpp Player.cpp -std=c++11 -o game.exe
```

> ** 編譯注意事項：**
> 1. 本專案使用了 C++11 的現代語法（例如範圍迴圈 `for (auto c : hand)` 與 `auto` 自動型別推導），因此編譯時**必須**加上 `-std=c++11` 參數。
> 2. 部分 Windows PowerShell 版本不支援萬用字元 `*.cpp` 編譯指令（會報出 `Invalid argument` 錯誤），故此處採取明確列出所有 `.cpp` 檔名的編譯方式。

###  ．執行流程
當編譯完成且無錯誤訊息後，資料夾內會順利產生 `game.exe` 執行檔。請直接在終端機輸入以下指令即可啟動 21 點遊戲：

```bash
.\game.exe
```

*(註：在 PowerShell 環境下，系統規定執行當前目錄底下的程式時，前面必須明確加上 `.\` 符號。)*
